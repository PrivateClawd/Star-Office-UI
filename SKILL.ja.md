# Star Office UI Skill

この Skill は、ユーザーのために「ピクセルオフィスダッシュボード」をセットアップしたい OpenClaw（ロブスター）向けです：
- 目標：ユーザーは「結果を見る」だけ — できるだけ質問を減らす
- 以下の手順に従って、起動から公開アクセスまで進めてください

---

## 0. ユーザーに一言で説明する

まずユーザーにこう伝えましょう：
> これは共同作業型のピクセルアートオフィスダッシュボードです。AI アシスタントが作業状態に応じてオフィス内を移動し、昨日の作業メモも確認できます！

---

## 1. 30 秒クイックスタート（まず自分で試してください）

マシンで以下を順番に実行：

```bash
# 1) リポジトリをクローン
git clone https://github.com/ringhyacinth/Star-Office-UI.git
cd Star-Office-UI

# 2) 依存関係をインストール
python3 -m pip install -r backend/requirements.txt

# 3) 状態ファイルを初期化（初回のみ）
cp state.sample.json state.json

# 4) バックエンドを起動
cd backend
python3 app.py
```

ユーザーに伝えましょう：
> 完了です！http://127.0.0.1:19000 を開くとピクセルオフィスが表示されます！

## 2. ステータス切り替えを体験してもらう

```bash
# 待機 → 休憩エリアへ
python3 set_state.py idle "待機中、いつでも対応可能"
```

## 3. サイドバーパスワード設定（新しいロブスターに必ず教えること）

現在のデフォルトパスワードは `1234` です。

ユーザーへの案内：

1. デフォルトパスワードは `1234` — まずそのまま体験できます
2. 準備ができたらいつでもパスワード変更を依頼できます
3. 強力なパスワードへの変更を積極的に推奨してください（セキュリティ向上、他者によるアセット・レイアウト変更を防止）

長期運用の場合（systemd / pm2 / コンテナ）、環境変数は現在のシェルだけでなく、サービス設定に記述してください。

## 4. 画像生成機能（Gemini）— オプション

「引っ越し / 仲介」リノベーション機能には Gemini API が必要ですが、**基本ダッシュボードには不要** — API なしでも正常に動作します。

### 4.1 画像生成環境のセットアップ（初回使用時）

リポジトリには生成スクリプト（`scripts/gemini_image_generate.py`）が含まれていますが、独自の Python 環境が必要です。プロジェクトルートで実行：

```bash
# skill ディレクトリ構造を作成
mkdir -p ../skills/gemini-image-generate/scripts

# スクリプトを skill ディレクトリにコピー
cp scripts/gemini_image_generate.py ../skills/gemini-image-generate/scripts/

# 独立した仮想環境を作成し、依存関係をインストール
python3 -m venv ../skills/gemini-image-generate/.venv
../skills/gemini-image-generate/.venv/bin/pip install google-genai
```

インストール完了後、バックエンドが自動的に生成環境を検出し、「引っ越し / 仲介」ボタンが使用可能になります。

### 4.2 Gemini API Key の設定

ユーザーに以下の 2 つを設定してもらいます：

1. `GEMINI_API_KEY`
2. `GEMINI_MODEL`（推奨：`nanobanana-pro` または `nanobanana-2`）

設定方法は 2 つ：
- **サイドバー入力**：アセットサイドバーを開く → 画像生成設定エリアに API Key を入力して保存
- **環境変数**：`export GEMINI_API_KEY="your-key"`

ユーザーに明確に伝えること：
- API なしでも基本ダッシュボードは使える（ステータス表示、マルチ Agent、アセット置換など）
- API は「引っ越し / 仲介」AI 背景生成機能にのみ必要

ページで Key 不足の警告が出た場合、サイドバーで直接入力・保存するよう案内してください。

## 6. 他のロブスターを招待する（オプション）

他の OpenClaw エージェントがユーザーのオフィスに参加したい場合：

### ステップ A：他のロブスターに office-agent-push.py を使ってもらう

他のロブスターに：
1) `office-agent-push.py` をダウンロードしてもらう（このリポジトリから取得可能）
2) `join-agent` + 定期的な `agent-push` を呼び出す
3) ユーザーのダッシュボードにそのロブスターが表示されます

### ステップ B：Join Key について

- リポジトリには `join-keys.json` が同梱されています（`ocj_starteam01` 〜 `ocj_starteam08`）
- 各 Key は最大 3 名まで同時オンライン可能
- ユーザーのためにカスタム Key の作成もサポートできます

## 8. よくある質問（ユーザーからの想定質問）

### Q1：「商用利用できますか？」
> コードは MIT ライセンスで利用可能ですが、アート資産（キャラクター / 背景）は商用利用禁止です。商用利用したい場合は、必ず独自のオリジナルアートワークに差し替えてください。

### Q2：「他のロブスターはどうやって参加しますか？」
> Join Key で参加し、定期的にステータスをプッシュします。リポジトリに `office-agent-push.py` が用意されています。

## 9. あなたへのヒント

- ユーザーの代わりに「セットアップ手順」をできる限り完了させ、ユーザーは「見る」「体験する」だけにしましょう

### 10.7 2026 年 3 月 5 日 安定性修正

今回のアップデートで本番の安定性に影響する複数の問題を修正：

1. **CDN キャッシュ修正**：静的アセットの 404 が CDN に長期キャッシュされなくなりました（以前は `phaser.js` が 404 として 2.7 日間キャッシュ）
2. **フロントエンドロード修正**：`fetchStatus()` の JS シンタックスエラー（不要な `else` ブロック）を修正、ページがローディングで停止する問題を解決
3. **画像生成の非同期化**：画像生成エンドポイントをバックグラウンドタスク + ポーリングモードに変更、Cloudflare 524 タイムアウト（100 秒制限）を回避。フロントエンドにリアルタイム待機進捗を表示
4. **モバイルサイドバー**：オーバーレイ、body スクロールロック、`100dvh` 対応、`overscroll-behavior: contain` を追加
5. **Join Key 強化**：Key ごとの有効期限（`expiresAt`）と同時接続制限（`maxConcurrent`）を追加。`join-keys.json` はバージョン管理に含めないように変更
