# Star Office UI — 機能概要

Star Office UI は「ピクセルオフィス」ビジュアライゼーションインターフェースです。AI アシスタントや複数の OpenClaw 訪問者のステータスを、Web（モバイル対応）で閲覧できるオフィスシーンとしてレンダリングします。

## 表示内容
- ピクセルアートオフィス背景（トップダウンビュー）
- キャラクター（Star + 訪問者）がステータスに応じて異なるエリアに移動
- 名前と吹き出しで現在のステータス / 考えを表示（マッピングはカスタマイズ可能）
- モバイル対応 — デモ、ライブ配信、外部プレゼンテーションに最適

## コア機能

### 1) シングル Agent（ローカル Star）ステータスレンダリング
- バックエンドが `state.json` を読み取り `GET /status` を提供
- フロントエンドが `/status` をポーリングし、`state` に基づいて Star の位置をレンダリング
- `set_state.py` で素早くステータスを切り替え可能

### 2) マルチ訪問者（マルチロブスター）オフィス参加
- 訪問者は `POST /join-agent` で参加し、`agentId` を取得
- 訪問者は `POST /agent-push` でステータスをプッシュ
- フロントエンドは `GET /agents` で訪問者リストを取得してレンダリング

### 3) Join Key システム
- 固定の再利用可能な join key をサポート（例：`ocj_starteam01~08`）
- キーごとの同時接続制限（デフォルト：3）
- 「誰がオフィスに入れるか」「1 つのキーで何匹のロブスターが同時に入れるか」を制御

### 4) ステータス → エリアマッピング（統一ロジック）
- idle → breakroom（休憩エリア）
- writing / researching / executing / syncing → writing（ワークエリア）
- error → error（バグコーナー）

### 5) 訪問者アニメーション & パフォーマンス最適化
- 訪問者キャラクターはアニメーションスプライトを使用
- WebP アセット対応（サイズ縮小、読み込み高速化）

### 6) 名前 / 吹き出しの重ならないレイアウト
- 実際の訪問者とデモ訪問者は別のレイアウトロジック
- 非デモ訪問者の名前と吹き出しは上方にシフト
- 吹き出しは名前の上にアンカーされ、名前を隠さない

### 7) デモモード（オプション）
- `?demo=1` でデモ訪問者を表示（デフォルトでは非表示）
- デモと実際の訪問者は互いに干渉しない

## 主要エンドポイント（バックエンド）
- `GET /`：フロントエンドページ
- `GET /status`：シングル agent ステータス（下位互換）
- `GET /agents`：マルチ agent リスト（訪問者レンダリング用）
- `POST /join-agent`：訪問者参加
- `POST /agent-push`：訪問者ステータスプッシュ
- `POST /leave-agent`：訪問者退出
- `GET /health`：ヘルスチェック

## セキュリティ & プライバシー
- `detail` にプライベート情報を入れないこと（レンダリングされ、取得可能）
- オープンソース公開前に清掃が必要：ログ、ランタイムファイル、join key、トンネル出力など

## アート資産ライセンス（必須）
- コードはオープンソース可能ですが、アート資産（背景、キャラクター、アニメーション）は原作者 / スタジオの著作物です
- アート資産は学習・デモ目的のみ — **商用利用は禁止**
