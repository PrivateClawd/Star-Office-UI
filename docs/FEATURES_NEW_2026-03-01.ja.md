# Star Office UI — 新機能（v2 リリース & PrivateClawd 統合）

## フェーズ 1：v2 リライト（2026-03-01）

### 1. マルチエージェント訪問者システム
- 複数のリモート OpenClaw エージェントが同一オフィスに同時参加可能。
- 訪問者は独立したアバター、名前、ステータス、エリア、吹き出しを持つ。
- 動的なオンライン/オフラインステータスとリアルタイム更新。
- 6 種類のゲストアバタースタイルとアニメーションスプライト（`guest_anim_1~6`）。

### 2. Join Key システムのアップグレード
- 「使い捨てキー」から「固定再利用可能キー」にアップグレード。
- デフォルトキー：`ocj_starteam01` 〜 `ocj_starteam08`。
- キーごとの同時接続制限 `maxConcurrent`（デフォルト：3）。
- `expiresAt` フィールドによるキー有効期限サポート。

### 3. 同時実行制御（レースコンディション修正）
- 同時 join 時のレースコンディションを修正。
- 同一キーでの 4 番目の同時 join が正しく拒否される（HTTP 429）。

### 4. 訪問者ステータスマッピング & エリアレンダリング
- `idle` → breakroom、`writing/researching/executing/syncing` → writing、`error` → error。
- 状態正規化エイリアス（例：`working` → `writing`、`run` → `executing`）。
- 訪問者の吹き出しテキストがステータスと同期 — ずれ解消。

### 5. 訪問者アニメーション & アセット最適化
- 訪問者を静止画からアニメーションスプライト（ピクセルアート風）にアップグレード。
- `guest_anim_1~6` を WebP 形式で提供、ファイルサイズ削減。
- ゲストロールアイコン：`guest_role_1~6`（PNG）。

### 6. 名前 & 吹き出し表示の最適化
- 非デモ訪問者の名前と吹き出しを上方にシフトし、キャラクターの重なりを防止。
- 吹き出しのアンカーポイントを名前ベースの位置決めに変更、「吹き出しは常に名前の上」を保証。

### 7. モバイル表示
- `max-width: 900px` または `pointer: coarse` でレスポンシブレイアウト。
- ゲームキャンバス：100vw × 66.666vh；コントロールパネル：100vw × 33.334vh。
- タッチフレンドリー：最小ボタン高さ 44px、2 カラムゲストグリッド。
- アセットドロワー：92vw、バックドロップオーバーレイとボディスクロールロック。

### 8. リモートプッシュスクリプトの改善
- 状態ファイルからの読み取りとオフィスへのステータスプッシュをサポート。
- ステータスソース診断ログを追加（「なぜいつも idle なのか」のデバッグ用）。
- `AGENT_NAME` 環境変数のオーバーライドタイミング問題を修正。

### 9. 3 言語 UI（i18n）
- 3 言語対応：中国語（zh）、英語（en）、日本語（ja）。
- `I18N` オブジェクトと `t(key)` 翻訳関数で全 UI 文字列を管理。
- 言語設定を `localStorage` に永続化。
- 右下の言語切り替えボタン。

## フェーズ 2：アセット管理 & AI 生成（2026-03-01 〜 03-05）

### 10. アセットドロワー（サイドバー）
- 右側パネル（320px 幅）、パスワード保護付き。
- 全アセットリスト、表示/非表示トグル、位置編集（X, Y, scale）。
- カスタムアセットのアップロードとアニメーションファイルの自動スプライトシート変換。
- `.default` スナップショットまたは `.bak` バックアップからのアセット復元。
- モバイル：92vw 幅、バックドロップオーバーレイ、スクロールロック。

### 11. AI 背景生成（Gemini）
- Gemini API を使用した RPG スタイルのピクセルアート背景生成。
- 2 つの速度モード：Fast（1152×648）と Quality（1280×720）。
- 8 つのランダムテーマ：ダンジョンギルドルーム、Stardew Valley 風酒場、北欧ファンタジー酒場、魔法技術ワークショップ、エルフの森の宿屋、ピクセルサイバー酒場、砂漠キャラバンの宿、雪山ロッジ。
- カスタムプロンプトで完全パーソナライズ背景をサポート。
- ワーカースレッドでの非同期生成により Cloudflare 524 タイムアウトを回避。
- モデルオプション：`nanobanana-pro` と `nanobanana-2`。

### 12. ホームお気に入り
- 生成またはカスタム背景を最大 30 個お気に入りとして保存。
- 適用、削除、現在の背景を保存の操作。

### 13. Gemini 設定 UI
- アセットドロワーから API キーとモデルを設定。
- セキュリティのためマスクされたキー表示。
- ランタイム設定は `runtime-config.json`（chmod 0600）に保存。

## フェーズ 3：セキュリティ & 安定性（2026-03-04 〜 03-06）

### 14. 本番セキュリティ強化
- 本番環境で `FLASK_SECRET_KEY` は ≥ 24 文字が必須。
- `ASSET_DRAWER_PASS` は ≥ 8 文字で `1234` 以外が必須。
- チェック失敗時は即座に起動停止。
- セッション設定：HttpOnly、SameSite=Lax、本番では Secure、12 時間有効期間。

### 15. CDN キャッシュ修正
- 404 レスポンスに明示的な `no-store` ヘッダーを追加し、CDN のキャッシュを防止。

### 16. バックエンドモジュールリファクタリング
- バックエンドをフォーカスされたモジュール（`memo_utils.py` 等）に分割し保守性向上。

### 17. 自動アイドル TTL
- 作業状態は `ttl_seconds`（デフォルト 300 秒）後に古くなった場合、自動的に `idle` にリセット。
- クラッシュ後にエージェントが永久に「ビジー」表示されることを防止。

### 18. ポート変更
- デフォルトポートを 18791 から 19000 に変更し、OpenClaw Browser Control のポート衝突を回避。

## フェーズ 4：PrivateClawd 統合（2026-03 〜）

### 19. プロキシによるマルチテナンシー
- Star Office は Docker Compose サイドカーサービスとして実行。
- Next.js 認証済みリバースプロキシ `/api/star-office/proxy/`。
- ステートエンドポイント（`/status`、`/agents`）をインターセプトし、PostgreSQL からユーザーごとのデータを返す。
- 書き込みエンドポイント（`/set_state`、`/agent-push`、`/join-agent`、`/leave-agent`）はノーオペレーション成功を返す。
- 静的アセット、設定、生成エンドポイントは Flask バックエンドに転送。

### 20. URL 書き換え
- HTML レスポンス内の全ルート相対 URL をプロキシプレフィックスに書き換え。
- パターン：`/static/` → `/api/star-office/proxy/static/`、`fetch('/` → `fetch('/api/star-office/proxy/`。

### 21. Cookie 転送
- プロキシが `cookie` ヘッダーを Flask に転送し、`set-cookie` ヘッダーを返す。
- iframe を通じたアセットドロワーセッション認証を有効化。

### 22. Prisma データモデル
- `StarOfficeAgent` モデル：`userId`、`botId`、`agentName`、`status`、`statusMessage`、`avatarKey`。
- `[userId, botId]` のユニーク制約。
- `StarOfficeStatus` 列挙型：`idle`、`writing`、`researching`、`executing`、`syncing`、`error`。

### 23. ボットステータス同期
- `syncStarOfficeAgent()` をデプロイ、開始、停止、エラー、ステータスドリフト時に呼び出し。
- `BotStatus` → `StarOfficeStatus` マッピング：`running` → `writing`、`deploying/starting/stopping` → `syncing`、`stopped` → `idle`、`error` → `error`。
- `draft` と `deleting` のボットは非表示（エージェントレコード削除）。

### 24. エージェントごとのオプトイン
- `star-office` スキルが有効なエージェントのみがオフィスに表示。
- スキルを持つエージェントがない場合はガイド付き空状態を表示。

### 25. ダッシュボード統合
- サイドバーナビゲーションリンク（「Star Office」、Building2 アイコン）。
- デプロイ済みエージェントを持つユーザー向けメインダッシュボードのクイック起動カード。
- `StarOfficeView` コンポーネント：ツールバー付き iframe（エージェント数バッジ、更新、全画面、新しいタブ）。

### 26. Docker 設定
- `Dockerfile.star-office`：Python 3.12-slim、Flask + Pillow。
- Docker Compose サービス：ヘルスチェック、ボリューム永続化、起動順序の `depends_on`。
- 環境変数パススルー：`STAR_BACKEND_PORT`、`FLASK_SECRET_KEY`、`ASSET_DRAWER_PASS`。

### 27. 昨日のメモ統合
- star-office ルートからの相対パス `../memory/YYYY-MM-DD.md` から読み取り。
- コンテンツサニタイズ：OpenID トークン、ファイルパス、IP アドレス、メールアドレス、電話番号を削除。
- 長いコンテンツのトランケーションとランダム名言の追加。
