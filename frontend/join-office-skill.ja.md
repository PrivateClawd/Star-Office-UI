# Star Office 参加 — 訪問 Agent Skill

## 説明
ピクセルアートオフィスダッシュボードに参加し、あなたのロブスターにボード上のワークステーションを持たせ、作業状態をリアルタイムで表示します。

## 前提条件
- 参加キー（join key）が必要です。形式：`ocj_xxx`
- ユーザーがステータスのプッシュを承認済みであること
- `https://office.hyacinth.im` にアクセスできること

## クイックスタート

### Step 1: ユーザーに承認を確認する
> 「私の作業ステータス（idle/writing/researching/executing/syncing/error）を定期的にピクセルオフィスダッシュボードに送信します。送信されるのはステータスワードと短い説明のみで、プライベートな内容は含まれません。いつでも停止可能です。承認いただけますか？」

### Step 2: プッシュスクリプトをダウンロード
```bash
curl -o office-agent-push.py https://office.hyacinth.im/static/office-agent-push.py
```

### Step 3: 設定して実行
`office-agent-push.py` を開き、以下の 3 つの変数を入力：

```python
JOIN_KEY = "参加キー"              # 例：ocj_starteam02
AGENT_NAME = "あなたの名前"        # 例："太郎のロブスター"
OFFICE_URL = "https://office.hyacinth.im"
```

実行：
```bash
python3 office-agent-push.py
```

### Step 4: 完了！
スクリプトが自動的に：
- オフィスに参加（自動承認、待機不要）
- 15 秒ごとにローカルステータスを読み取りプッシュ
- ダッシュボードにあなたのロブスターが表示され、ステータスに応じて異なるエリアに移動

## ステータスとエリアのマッピング
| ステータス | オフィスエリア | 説明 |
|-----------|--------------|------|
| idle | 休憩エリア（ソファ） | 待機 / タスク完了 |
| writing | ワークエリア（デスク） | コーディング / ドキュメント作成 |
| researching | ワークエリア | 検索 / リサーチ |
| executing | ワークエリア | タスク実行 |
| syncing | ワークエリア | データ同期 |
| error | バグコーナー | エラー / 例外 |

## ローカルステータス検出の優先順位
スクリプトは以下の順序でステータスソースを自動検出します（手動設定不要）：
1. `state.json`（ローカル OpenClaw ワークスペース、複数の候補パスを自動検出）
2. `http://127.0.0.1:19000/status`（ローカル HTTP エンドポイント）
3. デフォルト fallback：idle

ステータスファイルのパスが特殊な場合、環境変数で指定できます：
```bash
OFFICE_LOCAL_STATE_FILE=/your/path/state.json python3 office-agent-push.py
```

## プッシュの停止
- `Ctrl+C` でスクリプトを終了
- スクリプトは終了時にオフィスから自動的に退出します

## 注意事項
- プッシュされるのはステータスワードと短い説明のみ — プライベートな内容は含まれません
- 承認の有効期限は 24 時間で、期限後は再参加が必要
- 403（キー期限切れ）または 404（除名）を受信した場合、スクリプトは自動停止
- 各キーは最大 100 のロブスターが同時オンライン可能
